# 📊 Retail Store SQL Analysis

## 🔍 Overview  
This project analyzes retail store operations using SQL to uncover insights on revenue, customer behavior, and product performance. 
The analysis focuses on identifying high-value customers, top-performing categories, and sales trends to support data-driven business decisions.

---

## 🎯 Objectives  
- Analyze sales and revenue performance  
- Understand customer purchasing behavior  
- Identify high-performing product categories  
- Extract actionable insights using SQL  

---

## 🛠 Tools & Technologies  
- SQL (MySQL)  
- MySQL Workbench  

---

## 🗄 Database Structure  
The dataset consists of the following tables:  
- **Customers** – customer details  
- **Products** – product information  
- **Orders** – transactions  
- **Order_Items** – item-level details  
- **Payments** – payment records  

---

## ⚙️ Key SQL Concepts Used  
- SELECT, WHERE, DISTINCT, BETWEEN, IN, LIKE  
- Aggregations: COUNT, SUM, AVG, GROUP BY  
- JOINS: INNER JOIN, LEFT JOIN, RIGHT JOIN  
- Subqueries (scalar & correlated)  
- Set operations (UNION)  
- Functions: CONCAT, COALESCE, DATE  

---

## 📈 Key Insights  
- 💰 Total revenue: **~₹69L**  
- 🛒 Average order value: **~₹17K**  
- 📦 Electronics is the top-performing category  
- 👥 High customer engagement (all customers placed orders)  
- 📊 No unsold products → efficient inventory  

---

## 💡 Business Value  
- Identifies top-performing products and categories  
- Helps analyze customer behavior  
- Supports data-driven decision-making  
- Improves inventory and sales strategy
