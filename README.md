📊 Retail Store SQL Analysis

🔍 Overview
This project focuses on analyzing retail store data using SQL to extract meaningful insights on sales performance, customer behavior, and product trends.
The goal is to demonstrate how structured SQL queries can be used to support data-driven business decisions.

🎯 Objectives
Analyze sales and revenue performance
Understand customer purchasing behavior
Identify high-performing product categories
Extract actionable insights using SQL
🛠 Tools & Technologies
SQL (MySQL)
MySQL Workbench
🗄 Database Structure

The dataset consists of the following tables:

Customers – customer details
Products – product information and categories
Orders – order transactions
Order_Items – item-level order details
Payments – payment records

⚙️ Key SQL Concepts Used
SELECT, WHERE, DISTINCT, BETWEEN, IN, LIKE
Aggregations: COUNT, SUM, AVG, GROUP BY
JOINS: INNER JOIN, LEFT JOIN, RIGHT JOIN
Subqueries (scalar & correlated)
Set operations (UNION)
Data transformation functions (CONCAT, COALESCE, DATE)

📈 Key Insights
💰 Total revenue generated: ~₹69L
🛒 Average order value: ~₹17K
📦 Electronics category shows highest sales performance
👥 High customer engagement with all customers placing orders
📊 No unsold products, indicating efficient inventory turnover

💡 Business Value
Helps identify top-performing products and categories
Enables customer behavior analysis for better targeting
Supports data-driven decision-making in retail operations
Improves inventory and sales strategy planning
